Proyecto web con HTML, Bootstrap, ramas main, dev y publicación en GitHub Pages

1. Estructura del proyecto según la imagen

El proyecto tiene una estructura organizada para un sitio web de varias páginas:


taller3/
├── app/
│       ├── forms.html
│       ├── tables.html
│       └── github.html
├── assets/
│          ├── css/
│          ├── img/
│          └── js/
├── index.html
└── README.md
Elemento Descripción
app/forms.html Página destinada a formularios.
app/tables.html Página destinada a tablas.
app/github.html Página destinada a la información guía para el manejo en GitHub.
app/assets/css Carpeta para archivos de estilos CSS, si el proyecto los requiere.
app/assets/img Carpeta para imágenes del proyecto.
app/assets/js Carpeta para archivos JavaScript.
README.md Archivo README con documentación.
index.html Página principal del proyecto.
Nota: Esta guía usa únicamente Bootstrap para el diseño visual. No incluye estilos CSS personalizados.

2. Requisitos iniciales

  • Tener una cuenta activa en GitHub.
  • Tener instalado Git en el computador.
  • Tener instalado Visual Studio Code.
  • Abrir la carpeta del proyecto desde Visual Studio Code.

3. Configuración de Git desde Visual Studio Code

Abre la terminal integrada de Visual Studio Code:

Menú: Terminal → New Terminal

Configura tu nombre y correo:

git config --global user.name "Tu Nombre"
git config --global user.email "tu_correo@email.com"

Verifica la configuración:

git config --global --list

4. Inicializar Git en el proyecto

Ubícate en la carpeta raíz del proyecto, donde está el archivo index.html, y ejecuta: 

git init
git add .
git commit -m "Primer commit: estructura inicial del proyecto"
Este primer commit guarda la versión inicial del proyecto, incluyendo index.html, forms.html, tables.html y la carpeta assets.

5. Crear repositorio en GitHub

  1. Ingresar a GitHub.
  2. Hacer clic en New repository.
  3. Asignar un nombre al repositorio, por ejemplo: proyecto-bootstrap-adso.
  4. Seleccionar si será público o privado.
  5. Crear el repositorio sin agregar README si el proyecto ya tiene archivos locales.

6. Conectar proyecto local con GitHub

Copia la URL del repositorio creado y ejecuta:

git remote add origin https://github.com/usuario/proyecto-bootstrap-adso.git
git branch -M main
git push -u origin main
Cambia usuario y proyecto-bootstrap-adso por los datos reales del repositorio.

7. Manejo de ramas: main, dev y feature

Se recomienda usar esta estrategia:

main

Contiene la versión estable del proyecto.

dev

Contiene los cambios en desarrollo antes de pasarlos a producción.

feature

Se usa para trabajar funciones específicas, como formularios o tablas.

Crear rama dev

git checkout -b dev
git push -u origin dev

Crear rama para formularios

git checkout -b feature/forms

En esta rama se puede modificar el archivo app/forms.html.

git add .
git commit -m "Mejora página de formularios con Bootstrap"
git push -u origin feature/forms

Crear rama para tablas

git checkout dev
git checkout -b feature/tables

En esta rama se puede modificar el archivo app/tables.html.

git add .
git commit -m "Mejora página de tablas con Bootstrap"
git push -u origin feature/tables

8. Integrar cambios de las ramas

Integrar feature/forms en dev

git checkout dev
git merge feature/forms
git push origin dev

Integrar feature/tables en dev

git checkout dev
git merge feature/tables
git push origin dev

Pasar dev a main

git checkout main
git merge dev
git push origin main

9. Publicar con GitHub Pages

  1. Entrar al repositorio en GitHub.
  2. Ir a Settings.
  3. Seleccionar Pages.
  4. En Branch, seleccionar main.
  5. En carpeta, seleccionar /root si el archivo principal index.html está en la raíz.
  6. Guardar los cambios.
La URL del sitio tendrá una estructura similar a:
https://usuario.github.io/proyecto-bootstrap-adso/
Si el archivo principal estuviera dentro de la carpeta app, se debería validar la ruta de publicación o mover el index.html principal a la raíz del proyecto.

10. Rutas recomendadas en el proyecto

Desde index.html hacia las páginas internas

<a href="app/forms.html" class="btn btn-primary">Formularios</a>
<a href="app/tables.html" class="btn btn-success">Tablas</a>

Desde forms.html o tables.html hacia assets

<link href="assets/css/styles.css" rel="stylesheet">
<script src="assets/js/app.js"></script>
<img src="assets/img/logo.png" class="img-fluid" alt="Logo del proyecto">

Desde forms.html o tables.html hacia index.html

<a href="../index.html" class="btn btn-secondary">Volver al inicio</a>

11. Actividad práctica para aprendices

El aprendiz debe realizar las siguientes acciones:

  • Configurar Git en Visual Studio Code.
  • Inicializar el repositorio local.
  • Crear repositorio en GitHub.
  • Subir el proyecto a la rama main.
  • Crear la rama dev.
  • Crear una rama feature/forms o feature/tables.
  • Realizar cambios usando Bootstrap.
  • Publicar el proyecto con GitHub Pages.
